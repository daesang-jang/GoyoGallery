프로젝트: goyo gallery 웹사이트

작업 목적 (Artists 구조 개선):
- 현재 Artists.tsx 안에 작가 목록/상세 정보가 뒤섞여 있는 구조를 정리하고,
- “1 작가 = 1 상세 파일” 구조로 분리해서,
  1) Artists.tsx는 **리스트 전용 페이지**로,
  2) 각 작가는 **개별 상세 페이지**로,
  3) 이미지/데이터는 **작가별로 정리된 구조**로
  만들고 싶다.

중요 제약사항(반드시 지켜야 함):
1) 기존 라우터 URL 구조와 Navigation 링크는 최대한 유지한다.
   - 현재 /artists 같은 기본 경로는 바꾸지 말고,
   - 새로운 상세 경로를 추가할 때도 기존 동작이 깨지지 않도록 할 것.
2) 외부에서 이미 사용하는 URL이 있다면, 그 URL이 깨지지 않도록 주의할 것.
3) Tailwind, 기존 컴포넌트 스타일/톤은 유지하고, **구조만 개선**하는 방향으로 작업한다.

원하는 최종 구조:

1️⃣ 작가 데이터 중앙화
- shared/artists.ts 파일을 새로 만들어서, 작가 메타데이터를 한 곳에서 관리하고 싶다.
- 타입/구조는 아래 예시처럼 설계해 달라:

export interface ArtistMeta {
  id: string;          // 예: "ha-jungwoo"
  name: string;        // 한글 이름
  nameEn?: string;     // 영문 이름
  medium?: string;     // 회화, 설치 등
  thumbnail: string;   // 리스트용 썸네일 경로
}

export const artists: ArtistMeta[] = [
  // 현재 Artists.tsx / ArtistDetail 관련 코드를 참고해서,
  // 실제 사용 중인 작가 정보를 여기에 합리적으로 옮겨와 달라.
];

2️⃣ 작가 상세 개별 파일 구조
- client/src/pages/artists/ 디렉터리를 만들고,
  각 작가당 1개 상세 파일을 생성해 달라. (예시 네이밍)

  client/src/pages/artists/ArtistDetail_ha-jungwoo.tsx
  client/src/pages/artists/ArtistDetail_upcoming01.tsx
  ...

- 각 상세 파일에는:
  - 작가 이름(한/영)
  - 매체 정보
  - 작가 소개 텍스트
  - 대표 이미지 리스트
  등을 포함해서, 현재 프로젝트의 디자인 가이드를 최대한 유지해서 구현해 달라.

3️⃣ Artists.tsx 리팩터링 (리스트 전용 페이지)
- client/src/pages/Artists.tsx는:
  - shared/artists.ts의 artists 배열을 import 해서,
  - 전체 작가 목록을 카드 형태로 나열하는 **리스트 전용 페이지**로 바꿔 달라.
- 각 작가 카드를 클릭하면,
  해당 작가의 상세 페이지(ArtistDetail_<id>)로 라우팅되도록 수정해 달라.
- 기존에 Artists.tsx 안에 섞여 있는 상세 관련 JSX/데이터는
  개별 상세 파일로 옮기고, 리스트 쪽에는 남기지 않도록 정리한다.

4️⃣ 이미지/폴더 구조 정리
- 이미지 폴더 구조를 작가별로 정리해 달라. 예시:

  client/public/images/Artist/HaJungWoo/...
  client/public/images/Artist/Upcoming/uc01.png

- 이미 프로젝트에 존재하는 이미지 경로를 참고해서,
  가능한 한 현재 파일들을 재사용하되,
  새로 만드는 상세 컴포넌트에서 이 경로들을 일관되게 사용하도록 정리해 달라.

5️⃣ Router/Navigation
- App.tsx 또는 라우터 정의 파일에서:
  - /artists → Artists.tsx
  이 기본 매핑은 유지한다.
- 필요 시 작가 상세 경로를 추가할 수는 있지만,
  추가할 때는:
  - 기존 /artists 동작이 깨지지 않게 할 것
  - 새 URL 패턴(예: /artists/ha-jungwoo 등)은
    현재 코드 구조를 분석해서 가장 자연스러운 형태로 제안해 줄 것
- Navigation 컴포넌트에 있는 Artists 관련 메뉴 텍스트/URL은
  함부로 바꾸지 말고, 변경이 필요하면 먼저 나에게 설명해 달라.

작업 방식 요청:

1) 먼저 실제 파일 수정 전에,
   - 현재 Artist/ArtistDetail 관련 구조 요약
   - 변경 후 구조 요약
   - 예상 영향 범위 및 리스크 (없으면 “리스크 없음”이라고 명시)
   를 짧게 정리해서 보여줘라.
   (실제 코드 수정은 이 요약에 대해 내가 “승인”이라고 말한 뒤에만 진행.)

2) 내가 “승인”이라고 말한 이후에만
   실제 파일을 수정해라.

3) 각 단계마다:
   - 무엇을 수정했는지 1~2줄 설명
   - git diff 스타일로 핵심 변경 코드 보여주기
   를 해라.

4) Tailwind, 기존 컴포넌트 스타일, UI 톤은 최대한 유지하고,
   데이터/파일 구조를 중심으로 정리해라.

지금부터 위 조건을 기반으로,
1단계: “실제 수정 없이, Artists 관련 현재 구조 요약 & 변경 후 구조 & 리스크”만 먼저 제안해 줘.
